# Database Schema & Migrations

## Purpose

Authoritative source for the database schema, migrations, and seed data. Every table, column, index, and relation is defined and documented here.

## Ownership

- `prisma/schema.prisma` — Database schema
- `prisma/migrations/` — Migration history (auto-generated by Prisma)
- `prisma/seed.ts` — Development seed data

## Local Contracts

- All models must have indexes on filter, sort, and join columns
- Never use raw SQL in application code — use Prisma Client exclusively
- Every migration must be reversible or explicitly flagged as irreversible
- Seed data must represent real, current pricing — no placeholder data in committed seeds
- Composite unique constraints prevent duplicate entries (e.g., [modelId, providerId] on Pricing)
- Cascade deletes: Pricing and ModelCapability are deleted when their parent Model is deleted

## Work Guidance

- Run `npx prisma migrate dev --create-only` for manual review before applying migrations
- Use `npx prisma studio` to visually inspect data during development
- Seed data currencies are always USD per 1M tokens
- New capabilities are added as rows in the Capability table, not as boolean columns on Model

## Verification

- `npx prisma validate` — Schema is syntactically correct
- `npx prisma db seed` — Seed script runs successfully on a clean database
- Migrations apply cleanly with `npx prisma migrate dev`
